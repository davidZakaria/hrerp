## 2024-06-23 - Prevent Directory Traversal Bypass in Express.js Static File Serving
**Vulnerability:** A directory traversal vulnerability allowed users to access arbitrary files bypassing the `startsWith` prefix check. URL encoded paths (e.g., `%2e%2e%2f` instead of `../`) were successfully parsed by `path.join`, and partial folder matches (e.g., `/uploads/avatars-fake` starting with `/uploads/avatars`) bypassed the `startsWith` check, allowing access to sensitive files.
**Learning:** URL parameters in `req.params` and `req.path` are not fully decoded by default in Express routes using wildcards or regexes when resolving paths manually. Additionally, using `startsWith` on file paths without appending a directory separator (`path.sep`) can allow traversal to laterally named directories (partial folder match).
**Prevention:** Always decode user-provided path segments using `decodeURIComponent` before resolving them. When validating path boundaries with `startsWith`, ensure the comparison string always ends with `path.sep` to strictly match the directory, avoiding lateral path traversal (e.g. `normalizedPath.startsWith(uploadsDir + path.sep)`).
