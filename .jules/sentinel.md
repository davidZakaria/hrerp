## 2024-11-20 - Fix Partial Folder Match Bypass in Path Validation
**Vulnerability:** A directory traversal check using `normalizedPath.startsWith(uploadsDir)` is vulnerable to partial folder match bypasses. If the intended directory is `/uploads/avatars`, a requested path traversing out to a sibling directory named `/uploads/avatars-secrets` would produce a normalized path like `/uploads/avatars-secrets/file.txt`, which mistakenly passes the `.startsWith('/uploads/avatars')` check.
**Learning:** Checking directory boundaries with `startsWith` using string matching requires careful consideration of path separators. Without appending a trailing slash, any directory starting with the same prefix string is considered inside the target directory. The use of `path.sep` makes this more robust across platforms.
**Prevention:** Always append `path.sep` to the target directory when performing string boundary checks with `startsWith`, e.g., `normalizedPath.startsWith(targetDir + path.sep)`, or use a dedicated method/library for checking path containment.
