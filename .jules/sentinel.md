## 2024-07-02 - Path Traversal via Partial Directory Match Bypass
**Vulnerability:** The `protectedFileAccess` middleware in `server.js` used `normalizedPath.startsWith(uploadsDir)` to prevent path traversal. This check was vulnerable to partial directory matches. For example, a request with path `/uploads/avatars-malicious/file.txt` would bypass the check for the `avatars` directory because the string `/uploads/avatars-malicious/file.txt` starts with `/uploads/avatars`.
**Learning:** Using `startsWith()` on file paths without considering directory separators allows attackers to bypass boundary checks by appending arbitrary strings to directory names if they can create sibling directories on the filesystem.
**Prevention:** When validating that a path is within a specific directory using `startsWith()`, always append a directory separator (`path.sep`) to the base directory path (e.g., `uploadsDir + path.sep`). Additionally, allow an exact match to the base directory if access to the directory itself is intended.
